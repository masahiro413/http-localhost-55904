# TASKS.md — 実装計画

このファイルは [SPEC.md](SPEC.md) を実現するための実装手順を、ステップに分解したものである。
**学びが主目的**のため、各ステップに「🎯 学びポイント」を添える。
上から順に、1ステップずつ動かして確認しながら進めるとよい。

- チェックボックス `- [ ]` は、完了したら `- [x]` にして進捗を管理する。
- 各ステップの「受け入れ基準」を満たしたら次へ進む。
- 使用技術：素の HTML / CSS / JavaScript（ビルド不要）。詳細は [CLAUDE.md](CLAUDE.md) 参照。

---

## Step 0：プロジェクトの土台をつくる

- [ ] `index.html` を作成し、最小の HTML5 スケルトンを書く（`<!DOCTYPE html>`, `lang="ja"`, `meta charset` 等）。
- [ ] `style.css` と `app.js` を作成し、`index.html` から読み込む
      （`<link rel="stylesheet" href="style.css">` と `<script src="app.js" defer></script>`）。
- [ ] ブラウザで `index.html` を開き、CSS と JS が読み込まれていることを確認する
      （例：JS で `console.log("hello")` を出す）。

🎯 学びポイント：HTML / CSS / JS の3ファイル分離（関心の分離）と、`<script defer>` の意味。

**受け入れ基準**：`index.html` を開くと、CSS が効き、コンソールにログが出る。

---

## Step 1：静的な UI（カード型レイアウト）

- [ ] ヘッダー（アプリ名 `My Tasks`）、入力エリア（テキスト欄＋「追加」ボタン）、
      タスクリスト用の空コンテナを HTML に用意する。
- [ ] CSS で全体を中央寄せ・最大幅を設定し、余白をゆったり取る。
- [ ] タスク1件を表す「カード」の見た目を CSS で作る（角丸・柔らかい影・親しみやすい配色）。
      まずは HTML に**ダミーのカードを2〜3個**直書きして見た目を確認する。
- [ ] CSS カスタムプロパティ（変数）で色を定義し、`@media (prefers-color-scheme: dark)` で
      ダークテーマの値に切り替える。

🎯 学びポイント：CSS 変数、`box-shadow`／`border-radius`、`prefers-color-scheme` による OS 追従テーマ。

**受け入れ基準**：カードが並んで見え、OS のテーマをライト／ダークに切り替えると配色が変わる。

## Step 2：タスクの追加

- [ ] `app.js` で、タスクを保持する配列 `tasks`（`{ id, text, createdAt }` の配列）を用意する。
- [ ] リストを描画する関数 `render()` を作り、`tasks` から DOM を組み立てて表示する
      （ダミーの直書きカードは削除し、JS 描画に置き換える）。
- [ ] 「追加」ボタンのクリックと、入力欄での **Enter** の両方で追加できるようにする。
- [ ] 追加時のバリデーション：`text.trim()` が空なら追加しない。
      追加できたら入力欄を空にしてフォーカスを戻す。
- [ ] 新規タスクはリストの**末尾に追加**する（下に積む）。

🎯 学びポイント：配列と DOM の同期（状態→描画）、`addEventListener`、`keydown` での Enter 検出、`trim()`。

**受け入れ基準**：入力して追加すると、カードが末尾に増える。空入力では増えない。

---

## Step 3：localStorage による永続化

- [ ] `save()`：`localStorage.setItem("simple-todo.tasks", JSON.stringify(tasks))` を実装する。
- [ ] `load()`：起動時に `localStorage.getItem` → `JSON.parse` で復元する。
      値が無い／壊れている場合は空配列にフォールバックする（`try/catch`）。
- [ ] タスク配列を変更する操作（追加）のあとに `save()` を呼ぶ。
- [ ] 起動時に `load()` してから `render()` する。

🎯 学びポイント：`localStorage` の read/write、`JSON.stringify`／`parse`、壊れたデータへの防御（`try/catch`）。

**受け入れ基準**：タスクを追加してページを再読み込みしても、タスクが復元される。

---

## Step 4：完了 → 即消え ＋ Undo トースト

- [ ] 各カードに完了用のチェックボックス（またはボタン）を置く。
- [ ] 完了操作で、そのタスクを `tasks` から取り除き、`save()` → `render()` する（一覧から即消える）。
- [ ] 画面下部に **Undo トースト**を表示し、「元に戻す」ボタンを置く。
- [ ] `setTimeout` で **5秒**後にトーストを消す。時間切れになったら、その完了を確定する（復元不可）。
- [ ] 「元に戻す」を押したら、取り除いたタスクを元の位置に近い形で `tasks` に戻し、`save()` → `render()`。
- [ ] 次の完了操作が来たら、前のトーストの猶予は打ち切って確定する
      （直前タスクの情報を1件分だけ保持しておく）。

🎯 学びポイント：`setTimeout`／`clearTimeout` による一時状態、配列からの要素除去（`filter`）と復元、
「保留 → 確定 or 取り消し」という UX パターン。

**受け入れ基準**：完了でカードが消え、トーストの「元に戻す」で戻せる。5秒放置すると戻せなくなり、
再読み込みしても消えたまま。

---

## Step 5：削除 ＋ 確認

- [ ] 各カードに、完了とは別の「削除」ボタンを置く。
- [ ] 削除ボタンで確認（`window.confirm("このタスクを削除しますか？")` 等）を挟み、
      承認された場合のみ `tasks` から取り除いて `save()` → `render()`。
- [ ] 削除には Undo を付けない（確認済みのため）。

🎯 学びポイント：破壊的操作の前に確認を挟む設計、完了（気軽）と削除（慎重）の UX の作り分け。

**受け入れ基準**：削除ボタンで確認が出て、OK すると消える。キャンセルすると残る。

---

## Step 6：空状態・仕上げ・アクセシビリティ

- [ ] タスクが0件のとき、空状態メッセージを表示する（例：「タスクはありません。今日やることを追加しましょう」）。
- [ ] 各ボタンに `aria-label` を付け、キーボードだけでも操作できることを確認する。
- [ ] 余白・色・影・フォントを微調整し、「毎日開きたくなる」見た目に仕上げる。
- [ ] コードにコメントを添え、あとで読み返して分かるようにする（学習用途）。

🎯 学びポイント：空状態（Empty State）の UX、最低限のアクセシビリティ、読みやすいコードの整え方。

**受け入れ基準**：0件で案内が出る。キーボードのみでも追加・完了・削除ができる。

## 発展課題（任意）

コアが完成したあと、さらに学びたいときの追加テーマ。実装は必須ではない。

- [ ] **インライン編集**：タスク本文をダブルクリックするとその場で編集でき、
      Enter／フォーカス外れで確定する（空文字確定は無視）。
      🎯 `contenteditable` または input 差し替え、`blur`／`keydown` イベント。
- [ ] **手動並び替え**：ドラッグ＆ドロップでタスクの順序を入れ替える。
      🎯 HTML Drag and Drop API、配列の並び替えと `save()`。
- [ ] **テーマ切替トグル**：OS 追従に加えて、画面内のボタンでライト／ダークを手動切替し、
      選択を `localStorage` に保存する。
      🎯 `data-theme` 属性による上書き、設定の永続化。

## 総合の動作確認手順（受け入れテスト）

すべて実装できたら、以下を手動で通して確認する。

1. `index.html` をブラウザで開く。0件のとき空状態メッセージが出る。
2. タスクを2〜3件追加する（Enter でも追加できること）。空入力では増えないこと。
3. 1件を完了する → 一覧から即消える → トーストの「元に戻す」で戻る。
4. もう一度完了し、5秒放置 → 戻せなくなることを確認。
5. 1件を削除する → 確認ダイアログ → OK で消え、キャンセルで残る。
6. ページを再読み込みする → 残っているタスクが復元される。
7. OS のテーマをライト／ダークに切り替える → 配色が追従して変わる。
8. マウスを使わず、キーボードのみで 2〜5 を再度通せることを確認する。
