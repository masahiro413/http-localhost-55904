# CLAUDE.md

このファイルは、このリポジトリで作業する際の方針を Claude Code（および開発者）に共有するためのものである。
詳細な仕様は [SPEC.md](SPEC.md)、実装手順は [TASKS.md](TASKS.md) を参照する。

## プロジェクト概要

PC の Web ブラウザ上で動く、個人用のシンプルな ToDo アプリ。
機能は「タスクの**追加・完了・削除**」に絞る。

**本当の目的は「Web の基礎を素の状態で学ぶこと」**。
機能を増やすことではなく、少ない機能を **きれいで読みやすいコード** と
**毎日開きたくなる見た目** で作り切ることに価値を置く。

## 技術スタック

- **HTML / CSS / JavaScript のみ**（フレームワーク・ビルドツールなし）。
- データは **`localStorage`**（同一ブラウザ内）に保存する。サーバーは使わない。

### フレームワークを使わない理由

- DOM 操作・イベント・`localStorage` を「素の状態」で理解することが学習の目的だから。
- 依存パッケージが無く、数年後も壊れにくい（寿命が長い）ため。
- 本アプリの規模では、フレームワークの利点よりも学びやすさ・単純さが上回るため。
  （このトレードオフは [SPEC.md](SPEC.md) §8 に記録。）

## ディレクトリ構成

```
├── CLAUDE.md     このファイル（プロジェクト方針）
├── SPEC.md       確定仕様（機能・データ・UI/UX・懸念とトレードオフ）
```

※ 学習のため HTML / CSS / JS を3ファイルに分離している（関心の分離）。

## 実行方法

ビルド不要。次のいずれかで動く。

- `index.html` をブラウザにドラッグ＆ドロップして開く。
- 開発時は VS Code の「Live Server」拡張などでローカルサーバー経由で開く（自動リロードが効く）。

## コーディング方針

- 学習用途のため、**コメントを厚めに**書き、あとで読み返して理解できるようにする。
- 奇をてらわず、**王道のパターン**で書く（読みやすさ最優先）。
- 状態（タスク配列）を変更したら保存し、画面を描き直す、という一方向の流れを保つ。
- 変数・関数名は意味の分かる名前にする。

## 主要な設計決定（要点）

詳細は [SPEC.md](SPEC.md)。ここでは背骨だけ記す。

- **完了**：クリックで一覧から**即消える**。履歴は残さない。
  ただし完了直後の**数秒間だけ「元に戻す」トースト**を出す（誤操作の安全網）。
- **削除**：完了とは別操作。「間違い／不要」を取り除くもので、**実行前に確認を挟む**（慎重に）。
- **データ**：`localStorage` に保存。**このブラウザだけ**で完結する（同期しない）。
- **タスク**：**1行テキストのみ**。締め切り・優先度などは持たない。
- **テーマ**：OS 設定に自動追従（`prefers-color-scheme`）。
- **並び順**：新規タスクはリスト**下部に追加**。
- **編集**：コアには含めず、[TASKS.md](TASKS.md) の「発展課題（任意）」で扱う。
